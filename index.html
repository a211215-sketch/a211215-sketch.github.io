Welcome to my native Android development portfolio showcasing my journey from building structural layouts to deploying connected cloud solutions. This site tracks my laboratory progress and the creation of my health management application.

🥦 SDG 3: Good Health and Well-Being — Nutrio supports healthy lifestyles by simplifying everyday nutrition and metric tracking.

Laboratory Exercises

Lab 1: UI Foundation

Building basic layout primitives and learning UI item alignment using Jetpack Compose.

I learned to structure standard screens using rows, columns, and modifiers. It gave me a strong foundation for arranging components visually using Kotlin.

Lab 2: User Interaction

Implementing interactive components, forms, buttons, and state variables into layouts.

I mastered handling user inputs using state tracking tools like mutableStateOf. It helped me understand how dynamic value updates refresh layouts.

Lab 3: Material Design

Polishing screen styling, color rules, and cards according to Material tokens.

I implemented typography and custom shapes for consistent UI styling. It showed me how uniform guidelines improve application accessibility.

Lab 4: Navigation Architecture

Constructing multi-screen flows and safe routes using ViewModels to retain data states.

I built structured screen pathways and managed data operations safely inside ViewModels. This preserves vital interface properties during device rotations.

Lab 5: Room Database

Integrating internal storage engines using offline entities and database access objects.

I structured persistent data setups using abstract SQLite entities and specialized DAOs. Utilizing Room kept these database operations running smoothly.

Core Projects

Project 1: Nutrio (Local Framework)

Problem Statement: Users find it difficult to track their diet and workout logs smoothly without a reliable local interface.

An offline nutrition application that combines custom forms, navigation paths, and shared ViewModels.

I unified multiple screen pathways while maintaining clean data boundaries. Separating data logic from the UI design prevented performance stuttering.

Project 2: Nutrio (Cloud Connection)

Problem Statement: Isolated databases restrict access from multiple devices and limit real-time grocery barcode lookups.

An advanced system upgraded with remote database syncing, barcode API integrations, and location sensors.

I integrated live cloud synchronization by connecting Firestore with our database code. Managing live networks and sensors taught me to build strong validation logic.

Reflection & Learning Journey

I started my journey by mastering the fundamentals of UI layout and user interaction in Labs 1 and 2, which I then polished using Material Design principles in Lab 3. I elevated my skills further in Labs 4 and 5 by implementing seamless screen navigation and managing local data persistence with Room database. Finally, I successfully synthesized all these core concepts into Projects 1 and 2, transforming my technical knowledge into "Nutrio," a fully functional and well-structured nutrition app.
